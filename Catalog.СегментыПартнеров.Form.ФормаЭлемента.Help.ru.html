Сегмент партнеров

Справочник Сегменты партнеров предназначен для хранения списка всех сегментов партнеров, определенных в торговом предприятии. С помощью сегмента партнеров производится дополнительная классификация партнеров, которая используется при определении правил взаимодействий и типовых условий продаж, а также при проведении маркетингового анализа. Разделение партнеров на сегменты позволяет.

  • Регистрировать различные типовые условия продаж (цены, скидки) для каждого сегмента партнеров с помощью типовых соглашений.
  • Проводить по каждому сегменту партнеров отдельные маркетинговые акции.

Важно. Использование сегментов партнеров включается/отключается соответствующей функциональной опцией (Администрирование/Маркетинг и CRM).

Сегмент – это набор элементов, удовлетворяющих правилам, определенным для сегмента. Он предоставляет информацию по элементам этого набора в виде, соответствующем цели создания сегмента. Сегменты допускают пересечения (один элемент может входить в несколько сегментов). Правила формирования и формы представления сегментов могут использовать произвольную информацию, хранимую в системе. Поддерживаются три способа формирования сегментов.

  • Динамический – набор элементов не хранится в таблицах базы данных, хранятся только правила его формирования. Состав сегмента формируется автоматически при вызове состава сегмента из карточки сегмента.
  • Периодическое формирование – набор элементов хранится в таблицах базы данных. Состав сегмента формируется при нажатии на кнопку Сформировать или при выполнении регламентного задания Формирование сегментов по расписанию, определенному пользователем;
  • Формирование «вручную» – пользователь может добавить или удалить элементы набора сегмента (при этом предоставляется возможность предварительно сформировать набор элементов по правилам, определенным для сегмента).

Правила формирования сегмента задаются непосредственно в форме сегмента в отдельном диалоговом окне, которое открывается при нажатии на кнопку Редактировать. Пользователь может использовать шаблоны настроек (Схема компоновки данных), с помощью которых формируются правила формирования сегмента партнеров. Для сегмента партнеров предусмотрены следующие шаблоны настроек правил формирования сегмента.

  • Основная схема. С помощью этой схемы можно произвести отбор элементов из справочника Партнеры по группам партнерам, региону, дополнительным свойствам и т.д.
  • По взаимодействиям При использовании этой схемы к тем отборам, которые предусмотрены в основной схеме, добавляется отбор по показателям, связанными с взаимодействиями с партнером. Используя эти показатели можно например, сформировать сегмент партнеров по тем клиентам, с которыми давно не было взаимодействий и возобновить с ними отношения.
  • По продажам При использовании этой схемы к тем отборам, которые предусмотрены в основной схеме, добавляется отбор по показателям, связанным с продажей товаров партнерам: валовая прибыль,оборот, количество проданной номенклатуры, себестоимость и т.д. Используя эти показатели можно отслеживать выполнение условий продаж партнерами (например, сумма оборота за квартал больше 10000 рублей), автоматически исключать из сегмента партнеров, не выполняющих свои обязательства и соответственно применять к ним другие условия продаж (типовые соглашения).
  • По расчетам При использовании этой схемы к тем отборам, которые предусмотрены в основной схеме, добавляется отбор по показателям, связанным с взаиморасчетами с партнерами: долг партнера и наш долг. Используя эти показатели можно отслеживать список тех клиентов, которые должны нам большие суммы и тех поставщиков, по которым есть задолженность.

Важно. При формировании сегментов номенклатуры можно также использовать любые произвольные запросы. Для формирования запросов используется произвольная схема компоновки данных.

Работа с сегментом партнеров

Какая информация должна быть заполнена в карточке сегмента партнера?

  • Наименование - текстовое наименование сегмента. Наименование сегмента показывается в списках для выбора и используется для быстрой идентификации сегмента партнера.
  • Описание - подробное описание назначения сегмента партнера, правил его формирования.
  • Способ формирования - способ формирования сегмента.

Предусмотрено три способа формирования сегмента.

  • Формировать вручную - состав сегмента пользователь формирует вручную, есть возможность добавлять новых партнеров в сегмент, удалять имеющихся в сегменте. При создании сегмента вручную можно задавать правила, по которым будут отбираться партнеры в сегмент.
  • Периодически обновлять - состав сегмента вычисляется по правилам, указанным в настройках. Состав сегмента может обновляться пользователем вручную или с помощью регламентного задания. Регламентное задание может выполняться в соответствии с заданным периодом обновления (Расписание). Результаты вычисления (состав сегмента) сохраняются в информационной базе и используются до момента очередной итерации вычисления.
  • Формировать динамически - состав сегмента вычисляется каждый раз по правилам, которые для него заданы, в информационной базе результаты вычислений не хранятся.

После указания всех основных параметров нажмите на кнопку Записать. Настройка правил формирования сегмента производится только после того, как определены основные параметры сегмента (в частности способ формирования сегмента партнера).

Важно. После записи информации о сегменте изменить способ формирования сегмента нельзя. Динамические сегменты используются только для анализа информации. Динамические сегменты нельзя использовать при определении условий продаж (соглашения), условия по скидкам, ценам и т.д.

Можно ли при вводе нового партнера автоматически проверять выполнение необходимых условий для вхождения в сегмент?

Такая возможность предусмотрена для сегментов со способом формирования Периодически обновлять. Если при вводе нового партнера необходимо проверять принадлежность его к сегменту, то в карточке сегмента следует установить флажок Проверять на вхождение при создании нового партнера.

Пример. Сегменты партнеров разделены по бизнес-регионам. Для каждого бизнес-региона настроен свой сегмент с отбором по конкретному бизнес-региону. При вводе нового партнера и указания бизнес-региона, он будет автоматически отнесен к нужному сегменту.

Определение правил формирования сегмента партнеров

Как открыть форму для настройки правил формирования сегментов партнеров?
  • В карточке сегмента партнеров нажмите на кнопку Редактировать.
  • Будет открыта форма для настройки правил формирования сегмента партнера.
  • Для настройки правил формирования сегмента будет загружена основная схема, с помощью которой может быть произведена настройка правил формирования сегмента с использованием тех параметров, которые определены для партнера.

Как задать правила по которым будет сформирован состав сегмента партнера?

При настройке правил формирования сегмента используется система компоновки данных. Пользователю предоставляется упрощенная схема настройки. С помощью упрощенной схемы настройки пользователь может редактировать следующие настройки правил формирования сегмента.

  • Производить отбор данных в соответствии с параметрами, которые определены схемой компоновки сегмента (Отбор).
  • Задавать значение параметров, если это предусмотрено схемой компоновки сегмента (Параметры).

При формировании нового сегмента загружается основная схема компоновки данных для формирования состава сегмента партнеров. В поставку программу входят дополнительные упрощенные схемы настройки, которые поставляются в виде шаблонов.

Как сформировать сегмент партнеров?
  • Для формирования сегмента партнеров нажмите на кнопку Сформировать в карточке сегмента.
  • Будет сформирован список партнеров (состав сегмента) в соответствии с теми правилами, которые указаны в настройках формирования сегмента.

Важно. Динамические сегменты формируются без участия пользователя. Формирование сегмента запускается в момент просмотра состава сегмента.

Важно. Сегменты, для которых установлен способ формирования Периодически обновлять могут обновляться по расписанию с помощью регламентных заданий. Для каждого сегмента используется свое регламентное задание, расписание которого задается в форме сегмента. Список регламентных заданий для формирования сегментов можно посмотреть в разделе Администрирование/Поддержка и обслуживание (Регламентные и фоновые задания).

Как посмотреть и отредактировать состав сегмента?
  • В панели навигации формы сегмента выберите пункт Состав сегмента.
  • Появится список, в котором представлены данные текущего сегмента.
  • Набор полей, отображаемых в списке, определяется настройкой правил формирования сегмента.
  • Используя кнопку Обновить можно обновить данные в составе сегмента в соответствии с текущими данными информационной базы.

Важно. Редактирование данных в списке состава сегмента возможно только в том случае, если для сегмента партнеров задан способ формирования Формировать вручную. Для добавления информации о новых партнерах в список используется кнопка Добавить в сегмент, для удаления партнера из списка - Удалить из сегмента.

Можно ли в сегменте партнеров указать список тех конкурентов, которые работают с этим же сегментом клиентов?

С любым типом сегментов можно связать список партнеров-конкурентов, которые работают с тем же сегментом. Информация о конкурентах заполняется в табличной части Конкуренты.

Анализ данных по сегменту партнеров

Как проанализировать динамику продаж партнерами, входящими в сегмент?
  • В панели навигации карточки сегмента выберите пункт Продажи.
  • Будет сформирован отчет, в котором будет показана динамика продаж товаров партнерами, входящими в сегмент, распределенная по месяцам.

Как посмотреть информацию о том, какие партнеры данного сегмента относятся к другим сегментам?

Для этого используется отчет Пересечения сегмента партнеров, в котором показываются пересечения текущего сегмента со всеми остальными. При этом текущий сегмент и анализируемые сегменты должны быть нединамическими. Отчет вызывается из панели навигации карточки сегмента. В отчете отображается сгруппированная по сегментам информация.

  • Наименование сегмента.
  • Список партнеров, по которым сегмент пересекается с анализируемым сегментом.

Как посмотреть состояние взаиморасчетов с партнерами, которые относятся к данному сегменту?
  • В панели навигации формы сегмента выберите пункт Взаиморасчеты.
  • Будет сформирован отчет Расчеты с партнерами с отбором по тем партнерам, которые включены в сегмент.
